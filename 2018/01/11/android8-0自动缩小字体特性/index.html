android8.0自动缩小字体特性

andorid 官方在Android 8.0(api 26)中针对TextView出了一个新特性 自动收缩字体，这个特性对于一些涉及多语言的开发者来说还是很有用的，比如德国 俄罗斯这些国家的文字都是很长的,这些国家的文字相对普通国家的文字有可能字符数会翻一倍左右。这就可能出现 textView在大小写死的情况下或者字数太长的时候，会出现显示不下的情况，小屏手机就更加了，针对这种需求自动缩小字体就有用处了。

使用

  • 新特性记得api级别更新到26否则无法使用
  • xml

app:autoSizeTextType=”uniform” 使用特性
app:autoSizeMinTextSize=”12sp” 最小的size
app:autoSizeMaxTextSize=”100sp” 最大size
app:autoSizeStepGranularity=”2sp” 每次缩小多少

通过在xml中的设置自动缩小字体的特性就用上了，在字体超过限定大小时textView就会根据设置的 granularity值来进行缩小，有个地方要注意就是要使用AppCompatTextView的兼容view，因为这是新特性如果在api级别低的系统上面特性就会无效了。 官方列子给的是TextView来进行实例，实际使用中会有错误警告提示。

除了在xml中我们还可以通过在代码中来设置这些特性

我们还可以在arrays xml中设置预设大小来进行阶梯缩放
